Getting Started

Installation

No npm required. No webpack. No config. Include two files and you're building. 

<!-- In <head> -->
<link rel="stylesheet" href="https://unpkg.com/kern-ui@0.1.1/dist/kern.min.css">

<!-- Before </body> -->
<script src="https://unpkg.com/kern-ui@0.1.1/dist/kern.min.js"></script>
# Install
npm install kern-ui

# Import in your entry file
import 'kern-ui/kern.css';
import 'kern-ui/kern.js';
# Download and host yourself
kern.min.css  ~9KB gzipped
kern.min.js   ~3KB gzipped

# No CDN dependency, no SRI required
Design Philosophy

Three-layer contract

Kern has a clear mental model: HTML configures, CSS themes, JS controls.

⬡
HTML layer

data-variant, data-size, data-color, data-state — configure appearance declaratively, in markup.

◈
CSS layer

A full token system. Override --k-accent, --k-radius, --k-font-sans and the entire library repaints.

⟡
JS layer

Clean imperative API. Kern.toast(), Kern.dialog(), Kern.setAccent() — programmatic control when you need it.

Components

Accordion

Auto-closes other items by default. Add data-multi to allow multiple open at once.

Kern adds a proper variant system, a full JS API on every dynamic component, a documented CSS token system, and a clear mental model: HTML configures, CSS themes, JS controls. oat and Pico are simpler but less production-ready.
Most visual components (card, badge, alert, table, progress, avatar, breadcrumb, stepper) require zero JS. Dynamic components (tabs, dropdown, dialog, toast, drawer) are Web Components that enhance progressively.
Yes — perfectly. Kern's data-state, data-loading, and data-active attributes map directly to Alpine and htmx state, making server-rendered UIs trivially styled without framework overhead.

    <button data-accordion-trigger>Question</button>
    <div data-accordion-content>
      <div data-accordion-body>Answer text</div>
    </div>
  </div>
</div>

Built on native <dialog> for proper accessibility. Backdrop blur, focus trap, ESC to close. Trigger with data-toggle or Kern.dialog('id').

    </div>

    <div data-dialog-footer>...</div>

Components

Toast

Fully programmatic via Kern.toast(). Stacking, auto-dismiss, colored left-border accent, six positions. Pass a string for a quick message or an options object for full control.

// Minimal
Kern.toast('File uploaded successfully.');

// Full options
Kern.toast({
  title: 'Saved',
  message: 'Changes applied.',
  color: 'success',     // success | warning | danger | accent
  duration: 4000,       // ms — 0 = persistent
  position: 'bottom-right', // 6 positions
  dismissible: true
});

Design System

Design Tokens

Every visual property is a CSS custom property on :root. Override globally, or scope overrides to any selector. Component-specific tokens let you tune without touching globals.
